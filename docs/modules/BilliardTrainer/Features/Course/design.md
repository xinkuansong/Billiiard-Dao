> 文档最后更新：2026-02-27

## 设计目标与非目标

- **目标**：提供清晰的课程列表展示，按难度分组，明确显示锁定和完成状态，为后续课程内容集成预留接口
- **非目标**：不实现课程播放、训练执行、购买流程、进度持久化（当前为静态数据）

## 不变量与约束（改动护栏）

### 单位与坐标系

- 课程时长单位为分钟（Int）
- 课程 ID 从 1 开始递增，按等级连续编号

### 数值稳定性保护

- 课程数据为静态常量，不可在运行时修改（`let` 声明）
- `isCompleted` 为可变状态，但当前未持久化，需注意数据一致性

### 时序与状态约束

- 课程展示顺序固定：入门 → 基础 → 进阶 → 高级
- 锁定状态由 `isLocked` 参数控制，当前硬编码为 true/false

### 常量与阈值清单

| 常量/阈值 | 值 | 来源/理由 | 改动影响 |
|-----------|-----|----------|----------|
| 免费课程数量 | 3（L1-L3） | 产品规划 | 影响首页进度计算 |
| 基础课程数量 | 5（L4-L8） | 产品规划 | 影响购买包内容 |
| 进阶课程数量 | 6（L9-L14） | 产品规划 | 影响购买包内容 |
| 高级课程数量 | 4（L15-L18） | 产品规划 | 影响购买包内容 |
| 课程总数量 | 18 | 产品规划 | 影响统计和进度计算 |

## 状态机 / 事件模型

```
课程状态：
未锁定 + 未完成 → 可访问（显示播放图标）
未锁定 + 已完成 → 已完成（显示勾选图标）
已锁定 → 不可访问（显示锁定图标 + 降低透明度）
```

## 错误处理与降级策略

- 课程数据缺失：使用空数组，不显示对应分组
- 课程 ID 重复：依赖静态数据保证唯一性，运行时不做校验
- 锁定状态不一致：当前硬编码，后续需与购买状态同步

## 性能考量

- 静态数据加载：O(1)，无性能热点
- 列表渲染：SwiftUI 自动优化，ForEach 使用 Identifiable 提升效率
- 无网络请求或异步操作

## 设计决策记录（ADR）

### 使用静态数据而非 SwiftData 模型

- **背景**：课程内容相对固定，无需频繁变更
- **候选方案**：
  1. 静态常量（当前方案）
  2. SwiftData 模型 + 初始化数据
  3. JSON 配置文件
- **结论**：选择静态常量，简化实现，减少运行时开销
- **后果**：课程内容变更需修改代码并重新发布

### 锁定状态硬编码

- **背景**：购买流程未实现，暂时硬编码锁定状态
- **候选方案**：
  1. 硬编码（当前方案）
  2. 从 UserProfile 读取购买状态
  3. 从 StoreKit 读取购买状态
- **结论**：暂时硬编码，待购买模块完成后迁移
- **后果**：需后续重构以支持真实购买状态

## 影响面清单

改动本模块时，必须检查以下影响面：

- [ ] 调用方：ContentView（TabView 导航）
- [ ] 共享常量/状态：@AppStorage 键值（soundEnabled、hapticEnabled、aimLineEnabled、trajectoryEnabled）
- [ ] UI 交互链：设置开关切换、已购内容导航、帮助页面导航
- [ ] 持久化/数据映射：UserDefaults 存储、UserProfile 模型字段同步（当前未实现）
- [ ] 配置/开关：AudioManager 读取设置、Scene 层读取设置

## 最小回归门禁

每次改动后必须验证（不可跳过）：

### 主路径验证

- [ ] 打开设置 Tab，验证四个设置 Section 正常显示（游戏设置、购买管理、关于、反馈）
- [ ] 验证游戏设置开关：切换音效、震动、瞄准辅助线、轨迹预测，验证设置保存（重启应用后仍生效）
- [ ] 验证已购内容页面：点击"已购内容"，验证已解锁和未解锁课程包显示

### 相邻流程验证（至少 2 个）

- [ ] 验证设置生效：修改音效设置后，验证 AudioManager 读取到新值（可能需要重启应用）
- [ ] 验证帮助页面：点击"使用帮助"，验证基础操作和相机控制说明正常显示

## 常见问题排查

| 症状 | 可能原因 | 排查路径 |
|------|----------|----------|
| 设置修改不生效 | AudioManager 未同步读取 | 检查 AudioManager syncWithUserSettings 调用时机 |
| 设置值丢失 | @AppStorage 键名变更 | 检查键名一致性，验证 UserDefaults 存储 |
| 设置与 UserProfile 不一致 | 数据未同步 | 检查 UserProfile 字段与 @AppStorage 同步逻辑 |

## 未覆盖风险与待补测项

| 风险描述 | 等级 | 建议补测方式 |
|----------|------|-------------|
| AudioManager 设置同步不及时 | 中 | 添加 @AppStorage 观察者，实时同步到 AudioManager |
| UserProfile 与 @AppStorage 数据不一致 | 中 | 实现数据同步逻辑，或统一使用一种存储方式 |
| 购买流程未实现 | 高 | 集成 StoreKit，实现购买和恢复购买功能 |
| 设置键名不一致导致读取失败 | 高 | 代码审查保证键名一致性，或使用常量定义键名 |

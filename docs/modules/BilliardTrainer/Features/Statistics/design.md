# Statistics（统计）- 设计与约束

> 对应 README：[README.md](./README.md)
> 文档最后更新：2026-02-27

## 设计目标与非目标

- **目标**：清晰展示用户训练数据，提供多维度统计视图（总览、进球率、练习时长），为后续数据分析和可视化扩展预留接口
- **非目标**：不实现数据收集、数据计算、数据导出、高级数据分析（V1.2 技能雷达图为占位）

## 不变量与约束（改动护栏）

### 单位与坐标系

- 练习时长单位为小时（总览）和分钟（每日统计）
- 进球率使用百分比（0.0-1.0 的 Double，显示时转换为百分比）

### 数值稳定性保护

- 当前使用硬编码数据，需注意后续集成 UserStatistics 时的数据格式一致性
- 进球率计算需处理除零情况（当前硬编码，后续需添加 guard 保护）

### 时序与状态约束

- 统计数据读取应在视图出现时执行（当前硬编码，后续需 @Query 或 @Environment）
- 每周数据按周一到周日顺序展示

### 常量与阈值清单

| 常量/阈值 | 值 | 来源/理由 | 改动影响 |
|-----------|-----|----------|----------|
| 进球率颜色阈值（优秀） | ≥ 0.7（70%） | UI 设计 | 影响 AccuracyRow 颜色显示 |
| 进球率颜色阈值（良好） | ≥ 0.5（50%） | UI 设计 | 影响 AccuracyRow 颜色显示 |
| 柱状图高度系数 | 1.5 | UI 设计 | 影响 PracticeTimeCard 柱状图高度 |
| 技能雷达图版本标签 | V1.2 | 产品规划 | 占位功能标识 |

## 状态机 / 事件模型

```
统计数据展示：
数据加载中 → 显示占位数据（当前硬编码）
数据加载完成 → 显示真实统计数据（待集成）
数据为空 → 显示默认值（0 或空状态）
```

## 错误处理与降级策略

- UserStatistics 查询失败：显示默认值（0 或占位数据）
- 数据格式不一致：使用 guard 保护，提供默认值
- 除零错误：进球率计算需 guard totalShots > 0

## 性能考量

- 数据读取：当前硬编码 O(1)，后续 @Query 由 SwiftData 优化
- 列表渲染：SwiftUI 自动优化，ForEach 使用 Identifiable
- 柱状图计算：简单数学运算，无性能热点

## 设计决策记录（ADR）

### 使用硬编码数据而非实时查询

- **背景**：UserStatistics SwiftData 模型已存在，但当前视图未集成
- **候选方案**：
  1. 硬编码数据（当前方案）
  2. @Query 实时查询 UserStatistics
  3. ViewModel 层封装数据读取
- **结论**：暂时硬编码，待数据收集流程完善后集成 @Query
- **后果**：需后续重构以支持真实数据展示

### 技能雷达图占位实现

- **背景**：技能雷达图功能规划在 V1.2，当前仅占位
- **候选方案**：
  1. 占位视图（当前方案）
  2. 不显示该卡片
  3. 使用第三方图表库实现
- **结论**：显示占位视图，明确标识 V1.2 版本，保持 UI 一致性
- **后果**：V1.2 时需实现真实雷达图逻辑

### 柱状图使用简单 RoundedRectangle

- **背景**：练习时长分布使用简单柱状图展示
- **候选方案**：
  1. SwiftUI RoundedRectangle（当前方案）
  2. Swift Charts 框架
  3. 第三方图表库
- **结论**：使用 SwiftUI 原生组件，减少依赖，满足当前需求
- **后果**：如需更复杂图表（如折线图、饼图），需引入 Swift Charts

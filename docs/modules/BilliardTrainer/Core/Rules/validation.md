# Rules（规则系统）- 验证与回归

> 对应 README：[README.md](./README.md)
> 文档最后更新：2026-02-27

## 影响面清单

改动本模块时，必须检查以下影响面：

- [ ] 调用方：`FreePlayViewModel`（调用 `processShot()` 并读取游戏状态）
- [ ] 共享常量/状态：无（规则模块不依赖共享常量）
- [ ] UI 交互链：游戏状态显示、犯规提示、自由球提示、游戏结束提示
- [ ] 持久化/数据映射：无（游戏状态不持久化，每局重新开始）
- [ ] 配置/开关：无

## 最小回归门禁

每次改动后必须验证（不可跳过）：

### 主路径验证

- [ ] **开球流程**：初始状态 `waitingBreak` → 开球（有进袋）→ 确定花色 → 进入 `playing(group)` → 状态消息正确
- [ ] **击球流程**：`playing(group)` 状态下击球 → 打进己方球 → 返回 `true`（继续击球）→ 状态更新正确
- [ ] **清台流程**：`playing(group)` 状态下打进最后一颗己方球 → 进入 `eightBallStage` → 状态消息提示打8号球
- [ ] **打8号球流程**：`eightBallStage` 状态下合法打进8号球 → 进入 `gameOver(won: true)` → 状态消息显示获胜

### 相邻流程验证（至少 2 个）

- [ ] **犯规流程**：击球犯规（如白球落袋）→ `processShot()` 返回 `false`（不继续击球）→ `isBallInHand = true` → 状态消息显示犯规类型
- [ ] **花色确定流程**：`openTable` 状态下打进全色球 → 确定花色为 `solids` → 进入 `playing(.solids)` → 状态消息显示"你的花色：全色球 (1-7)"
- [ ] **8号球犯规**：花色未定时打进8号球 → 进入 `gameOver(won: false)` → 状态消息显示"花色未定时打进8号球，输了"
- [ ] **球名格式兼容**：使用 `ball_N` 和 `_N` 两种格式的球名 → 规则判定正确 → 花色分组正确

## 可观测性

- 日志前缀：`[Rules]`
- 关键观测点：
  - `EightBallGameManager.phase`：当前游戏阶段
  - `EightBallGameManager.playerGroup`：当前玩家球组
  - `EightBallGameManager.isBallInHand`：是否自由球
  - `EightBallGameManager.statusMessage`：状态消息（用于 UI 显示）
  - `ShotResult.legal`：击球是否合法
  - `ShotResult.fouls`：犯规列表
- 可视化开关：无（规则判定结果通过状态消息显示）

## 常见问题排查

| 症状 | 可能原因 | 排查路径 |
|------|----------|----------|
| 花色判定错误 | 球名解析失败或球号范围检查错误 | 检查 `extractBallNumber()` 逻辑、检查球号范围 |
| 8号球规则判定错误 | 阶段判断错误或清台检查错误 | 检查 `remainingSolids/Stripes` 更新、检查阶段转换逻辑 |
| 犯规未检测到 | 事件列表为空或事件类型不匹配 | 检查 `GameEvent` 格式、检查 `analyzeShotEvents()` 逻辑 |
| 状态转换异常 | 状态机逻辑错误或事件顺序错误 | 检查 `processShot()` 逻辑、检查事件顺序 |
| 自由球状态错误 | `isBallInHand` 更新逻辑错误 | 检查犯规处理逻辑、检查 `ballInHandBehindLine` 设置 |

## 未覆盖风险与待补测项

| 风险描述 | 等级 | 建议补测方式 |
|----------|------|-------------|
| 事件顺序错误可能导致首碰球判定错误 | 中 | 添加事件顺序测试用例 |
| 球名格式变更可能导致解析失败 | 中 | 添加球名格式兼容性测试 |
| 并发访问状态可能导致状态不一致 | 低 | 确保单线程访问或添加锁保护 |
| 极端情况（如同时打进多颗8号球）未处理 | 低 | 添加极端情况测试用例 |
| 开球线后限制未实现 | 低 | 如需要，添加开球线后限制逻辑 |

## 测试用例

### TC-M-001: 单球直线运动

**目的**: 验证球在无碰撞情况下的直线运动轨迹

**前置条件**:
- 台面上仅放置一颗球
- 球杆瞄准正中（无偏移）

**操作步骤**:
1. 将球放置在台面中央
2. 用中等力度沿长轴方向击球
3. 观察球运动至自然停止

**预期结果**:
- 球沿直线运动，不偏转
- 运动过程平滑，无跳跃
- 逐渐减速直到停止

**判定标准**:
- PASS: 直线运动，减速自然
- FAIL: 明显偏转或运动不连续
- DEVIATION: 轻微偏转（<1度），可能是浮点精度问题

### TC-M-002: 球-球正面碰撞

**目的**: 验证两球正面碰撞后的速度传递

**前置条件**:
- 台面上放置两颗球，沿长轴对齐

**操作步骤**:
1. 母球放置在底端中线
2. 目标球放置在顶端中线
3. 用中等力度正面击打母球
4. 观察碰撞后两球运动

**预期结果**:
- 碰撞后母球几乎停止（或缓慢前移）
- 目标球沿原方向继续运动
- 无侧向偏移

**判定标准**:
- PASS: 速度传递自然，母球近乎停止
- FAIL: 目标球不动 / 母球穿过目标球 / 碰撞方向明显错误
- DEVIATION: 母球有轻微残余速度（正常现象，摩擦碰撞特性）

### TC-M-003: 库边反弹

**目的**: 验证球-库边碰撞的反弹角度

**操作步骤**:
1. 将球放置在台面中央偏下
2. 以约 30° 角击向长边库
3. 观察反弹角度

**预期结果**:
- 反弹角约等于入射角（考虑摩擦损耗会略小）
- 反弹速度低于入射速度

**判定标准**:
- PASS: 反弹角合理（入射角 ±10°）
- FAIL: 球穿过库边 / 反弹角明显不合理 / 球卡在库边
- DEVIATION: 反弹角偏差较大但方向正确

### TC-M-004: 旋转球（加塞）效果

**目的**: 验证 cue tip offset 产生的旋转效果

**操作步骤**:
1. 将球放置在台面下方中线
2. 击球时设置左偏移（left english）
3. 观察球运动路径

**预期结果**:
- 球应先直线运动（sliding 阶段）
- 然后逐渐向右弯曲（rolling 阶段，左旋产生右弧线）

**判定标准**:
- PASS: 弧线方向正确，弯曲程度合理
- FAIL: 无弧线 / 弧线方向错误 / 球直接侧飞
- DEVIATION: 弧线方向正确但幅度偏大或偏小

### TC-M-005: 多球开球场景

**目的**: 验证多球同时碰撞的稳定性和性能

**操作步骤**:
1. 设置标准开球阵型（三角阵列）
2. 用大力度击打母球
3. 观察碰撞扩散效果和帧率

**预期结果**:
- 球阵自然扩散
- 无球体穿越或重叠
- 帧率保持流畅 (≥30fps)

**判定标准**:
- PASS: 碰撞自然，无穿越/重叠，帧率流畅
- FAIL: 球体穿越/爆炸/卡住
- DEVIATION: 偶尔轻微重叠但自动恢复
